# Bolão Copa 2026 — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A página web (GitHub Pages) com a classificação do bolão, atualizada sozinha todo dia por um robô (GitHub Actions) que busca os placares oficiais da Copa 2026 e recalcula tudo.

**Architecture:** Repositório público no GitHub guarda a planilha mestre + código Python. Um workflow do GitHub Actions roda diariamente: busca placares no endpoint público da ESPN → preenche a aba `Resultados` → calcula pontuação/ranking em Python → reordena a classificação na planilha → gera `site/index.html` → publica no GitHub Pages e commita a planilha atualizada.

**Tech Stack:** Python 3.12+ (stdlib `urllib`, `datetime`, `unicodedata`, `json`), `openpyxl` (única dependência externa). GitHub Actions + GitHub Pages. Sem LibreOffice, sem pandas, sem chave de API.

## Global Constraints

- Linguagem: Python 3.12+ (CI usa `actions/setup-python` com 3.12).
- Dependência externa única: `openpyxl`. Tudo mais é stdlib. Não usar pandas nem LibreOffice.
- Fonte de placares: `https://site.api.espn.com/apis/site/v2/sports/soccer/fifa.world/scoreboard?dates=YYYYMMDD` (sem chave).
- Fórmula de pontos por jogo (espelhar EXATAMENTE a coluna K da planilha): só pontua se palpite e resultado forem ambos números; `+2` se `sign(pC−pF)==sign(rC−rF)`; `+1` se `pC==rC`; `+1` se `pF==rF`; `+1` se `pC+pF==rC+rF`. Máx 5.
- Bônus fixo `+10` somado aos totais de **Caio**, **Camps** e **Kim**.
- 13 participantes (nome de exibição → nome da aba): Beda→`Beda`, Pedro→`Pedro`, Mauro→`Mauro`, Rodrigo→`Rodrigo`, Paulo→`Paulo`, Su→`Su`, Biral→`Biral`, Mané→`Mané`, Romanelli→`Romanelli`, AH→`AH`, Caio→`Caio`, Camps→`Camps ` (com espaço no fim), Kim→`Kim`.
- Jogos nas linhas 5–76 das abas; colunas: A=nº, B=grupo, C=time1, D=gols1, F=gols2, G=time2, H=data, J=hora, K=pontos.
- Privacidade: a cópia da planilha no repo tem o contato pessoal removido (limpar `B2`, `B3`, `G3` de todas as abas).
- Match de jogos ESPN↔planilha é por **times** (não só data), por causa do fuso (jogos 22h BRT viram dia seguinte em UTC). Nunca sobrescrever um resultado já preenchido.
- Repo remoto: `https://github.com/iStockk/bolao-copa-2026.git`. Pages URL: `https://istockk.github.io/bolao-copa-2026/`.

```python
def _num(x):
    return isinstance(x, (int, float)) and not isinstance(x, bool)

def _sign(n):
    return (n > 0) - (n < 0)

def pontos_jogo(pc, pf, rc, rf):
    if not all(_num(v) for v in (pc, pf, rc, rf)):
        return 0
    p = 2 if _sign(pc - pf) == _sign(rc - rf) else 0
    p += 1 if pc == rc else 0
    p += 1 if pf == rf else 0
    p += 1 if pc + pf == rc + rf else 0
    return p
```

```yaml
name: Atualizar bolão
on:
  schedule: [{cron: "0 13 * * *"}]
  workflow_dispatch:
permissions:
  contents: write
  pages: write
  id-token: write
concurrency:
  group: pages
  cancel-in-progress: false
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with: {python-version: "3.12"}
      - run: pip install -r requirements.txt
      - run: python gerar.py
      - name: Commit planilha/estado atualizados
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add -A
          git commit -m "chore: atualização automática $(date -u +%F)" || echo "sem mudanças"
          git push
      - uses: actions/configure-pages@v5
      - uses: actions/upload-pages-artifact@v3
        with: {path: site}
  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment: {name: github-pages, url: "${{ steps.deployment.outputs.page_url }}"}
    steps:
      - id: deployment
        uses: actions/deploy-pages@v4
```
